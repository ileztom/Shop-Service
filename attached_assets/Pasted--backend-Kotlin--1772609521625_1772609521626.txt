Создать backend-сервис на Kotlin, который позволяет пользователям:

регистрироваться и авторизироваться
просматривать список товаров
делать заказы
отменять заказы
видеть свою историю покупок

Администратор:

добавляет товары
изменяет данные о товарах
получает статистику через закрытый API

Технически проект должен включать:

Ktor backend
PostgreSQL
Exposed ORM
JWT авторизация
Redis кэширование
RabbitMQ или Kafka (одно из)
Документация API (Swagger)
Unit + Integration + E2E тесты
Docker и docker-compose
Деплой в Railway / Render / DigitalOcean
Структура API

Аутентификация
POST /auth/register
POST /auth/login

Пользовательские маршруты
GET /products
GET /products/{id}
POST /orders
GET /orders
DELETE /orders/{id}

Админские маршруты
POST /products
PUT /products/{id}
DELETE /products/{id}
GET /stats/orders (только admin)

Функциональные требования

Работа с БД
Создать таблицы:

users
products
orders
order_items
audit_logs

Использовать:

связи 1:N
внешние ключи
индексы

Бизнес-логика
При создании заказа:

проверять наличие товара
уменьшать stock
писать запись в audit logs
отправлять сообщение в очередь (RabbitMQ/Kafka)
кэшировать данные заказа

Асинхронность и фоновые задачи
Реализовать worker, который:

читает сообщения из очереди
логирует события
отправляет email-заглушку (фейковую)

Обработка ошибок
корректные HTTP-коды
бизнес-ошибки: нет товара, нет доступа
логирование ошибок

Swagger документация
Каждый маршрут должен иметь:

описание
параметры
примеры
схемы моделей

Тестирование
Минимум:

3 unit-теста
2 integration-теста с TestContainers
2 e2e теста на API

Docker и CI/CD
Нужно:

Dockerfile
docker-compose для приложения + PostgreSQL + Redis
workflow GitHub Actions (сборка + тесты)

Деплой
Опубликовать в Railway или Render.
Предоставить:

ссылку на работающий сервис
инструкцию к запуску
